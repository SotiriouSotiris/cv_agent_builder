# CV Templates

Choose one structure before generating a CV. Each template is self-contained, single-column, and designed for reliable ATS text extraction.

| Option | File | Structure | Style | Best suited to |
| --- | --- | --- | --- | --- |
| Classic Monochrome | `classic-monochrome.tex` | Profile, Experience, Education, Skills | Traditional black and grey | Conservative employers and broadly applicable CVs |
| Modern Blue | `modern-blue.tex` | Profile, Selected Impact, Experience, Education, Skills | Clean sans serif with blue accents | Product, engineering, and modern business roles |
| Technical Teal | `technical-teal.tex` | Technical Summary, Technical Skills, Experience, Projects, Education | Compact teal technical hierarchy | Engineering roles where tools and projects matter early |
| Leadership Burgundy | `leadership-burgundy.tex` | Leadership Profile, Core Strengths, Leadership Experience, Earlier Experience, Education | Restrained burgundy accents | Senior, lead, founder, and management-oriented roles |

Each `.pdf` file is a compiled visual preview of the matching `.tex` source. When using a template, copy its structure into the target application's `cv.tex`, replace every placeholder with verified evidence, and apply `cv-language.md`.

Treat section order as a strong default rather than a rigid rule. Move `Education` earlier for a recent graduate when it is the strongest relevant evidence; lead with `Experience` for an established professional. Remove unused sections, keep the most relevant evidence early, and do not reduce the font or margins merely to force additional content onto one page.
